Lesson 1 · Foundations

What is a Tree?

A tree is a non-linear hierarchical data structure — a connected acyclic graph.

Core Definition

A tree is an abstract data type (ADT) that stores elements hierarchically. It is a connected, acyclic (no cycles) collection of nodes and edges.

With the exception of the top element (the root), every element has exactly one parent and zero or more children.

🌳
Key insight: Unlike arrays, linked lists, stacks or queues (all linear), trees let you model real-world hierarchies — file systems, org charts, expression trees, HTML DOM. Linear structures can only express "before/after" — trees express "above/below".
Why Not Linear Structures?

Linear structures like arrays, linked lists, stacks and queues store data sequentially. Operations take O(n) time as data grows — not acceptable in modern systems.

Trees allow quicker access and power critical algorithms in:

  • File systems (directories & files)
  • Graphical User Interfaces
  • Databases (B-trees, indexing)
  • Websites (DOM)
  • Compilers (syntax trees)
Real-World Scenarios

Scenario 1 — File System: Root directory → sub-directories → files. You can't model this in an array.

Scenario 2 — Company Hierarchy: CEO → CTO, CFO, CMO → their subordinates. Perfect tree structure.

Scenario 3 — Family Tree: The main terminology (parent, child, ancestor, descendant) literally comes from family trees.

Mnemonics to remember: Think of it as an upside-down real tree. Root is at the TOP. Leaves are at the BOTTOM. Relationships are PARENT → CHILD going downward.
Lesson 2 · Foundations

Tree Terminology

Every term you must know — with precise definitions.

Node
Entity holding a value + pointers to children. Last nodes (no children) = leaf/external nodes.
Root
Topmost node. Has NO parent. Every traversal starts here.
Edge
Link between two nodes. A tree with n nodes has exactly n-1 edges.
Leaf / External Node
Node with NO children. No outgoing edges below it.
Internal Node
Node with at least one child. All non-leaf nodes.
Parent
Node that has a branch going to another node below it.
Sibling
Nodes sharing the same parent. At same level, same branch.
Ancestor
Parent, grandparent, great-grandparent, etc. going UP the tree.
Descendant
Children, grandchildren, etc. going DOWN from a node.
Depth of Node
Number of edges from ROOT to the node. Root depth = 0.
Height of Node
Number of edges from node to deepest leaf BELOW it.
Height of Tree
Height of the root = depth of deepest node in tree.
Level
Root = Level 0. Its children = Level 1. Increments by 1 each step down.
Subtree
A node v plus ALL its descendants. Any node can be the root of a subtree.
Forest
Collection of DISJOINT trees. Remove the root of a tree → you get a forest of subtrees.
Path
Sequence of nodes where every consecutive pair forms an edge.
Height vs Depth — The Classic Confusion

Depth is measured from the ROOT downward. Height is measured from the node upward to the deepest leaf.

For a leaf node: height = 0, depth = distance from root.

For the root: depth = 0, height = height of the whole tree.

⚠️ Exam trap: Height of a single node = 0 (some books say 1). Know which convention your course uses. The slides use: height = number of edges to deepest leaf (so single node = 0).
Lesson 3 · Foundations

Tree vs Linked List

They share DNA — but trees are the evolved form.

The Connection

A tree has common behaviour with a Linked List because:

  • In a linked list we need to know the head — from head we traverse everything.
  • In a tree we need to know the root — from root we traverse everything.

The fundamental difference: in a singly linked list each node stores a link to one node. In a tree, each node stores addresses of many nodes (its children).

PropertySingly Linked ListTree
StructureLinearHierarchical
Entry pointHead pointerRoot pointer
Node children1 (next)0 to many
CyclesPossible (circular LL)Never (acyclic by definition)
SearchO(n)O(log n) in balanced
Parent pointerNone (singly)Implicit (you know who called you)
💡
Think of it this way: A linked list is a degenerate tree where every node has exactly one child. A tree is the generalization. That's why the C++ node code looks very similar — just replace next* with vector<Node*> children.
Lesson 4 · Implementation

General Tree in C++

Node design, construction, and recursive printing.

The TreeNode Class

We use a template class so the tree can hold any data type. Each node holds its data and a vector of pointers to its children.

C++ — TreeNode
// ── TreeNode class ──────────────────────────────────
template<typename T>
class TreeNode {
public:
    T data;
    vector<TreeNode<T>*> children;   // unlimited children

    TreeNode(T data) {
        this->data = data;
    }

    // Destructor: recursively delete all children
    ~TreeNode() {
        for (int i = 0; i < children.size(); i++) {
            delete children[i];   // calls child destructor recursively
        }
    }
};
🔑
Why vector<TreeNode*>? Because we don't know at compile time how many children each node will have. A vector lets us dynamically add any number of child pointers. This is what makes it a general tree (not limited to 2 children like Binary Trees).
Print Tree Recursively
C++ — printTree
void printTree(TreeNode<int>* root) {
    if (root == NULL) return;  // base case / edge case

    cout << root->data << ": ";

    // Print all children on same line
    for (int i = 0; i < root->children.size(); i++) {
        cout << root->children[i]->data << " ,";
    }
    cout << endl;

    // Recursively print each child's subtree
    for (int i = 0; i < root->children.size(); i++) {
        printTree(root->children[i]);
    }
}
🔁
Recursion pattern: Base case first (NULL check). Then do work on current node. Then recurse on children. This is preorder style — process node BEFORE children. Almost all tree recursive functions follow this pattern.
Lesson 5 · Implementation

Level Order Input & Print

How to build a tree from user input using a queue — and print it level by level.

Taking Input Level-Wise (General Tree)

We use a queue to process nodes level by level. For each node we dequeue, we ask the user how many children it has, then create those children and enqueue them for future processing.

C++ — takeInputLevelWise
TreeNode<int>* takeInputLevelWise() {
    int rootData;
    cout << "Enter root data: ";
    cin >> rootData;

    TreeNode<int>* root = new TreeNode<int>(rootData);
    queue<TreeNode<int>*> q;
    q.push(root);

    while (!q.empty()) {
        TreeNode<int>* f = q.front();
        q.pop();

        cout << "Enter no of children of " << f->data << ": ";
        int n;
        cin >> n;

        for (int i = 1; i <= n; i++) {
            int childData;
            cout << "Enter child " << i << " of " << f->data << ": ";
            cin >> childData;
            TreeNode<int>* child = new TreeNode<int>(childData);
            q.push(child);           // enqueue for later processing
            f->children.push_back(child);  // connect to parent
        }
    }
    return root;
}
Level Order Print (BFS Print)
C++ — printTreeLevelWise
void printTreeLevelWise(TreeNode<int>* root) {
    queue<TreeNode<int>*> q;
    q.push(root);
    int counter = 0;  // counts total nodes visited

    while (!q.empty()) {
        TreeNode<int>* f = q.front();
        counter++;
        q.pop();

        cout << f->data << ": ";
        for (int i = 0; i < f->children.size(); i++) {
            cout << f->children[i]->data << " ,";
            q.push(f->children[i]);   // enqueue children for next level
        }
        cout << endl;
    }
    cout << "Total nodes: " << counter << endl;
}

// Key insight: print node FIRST, then enqueue children
// This guarantees level-by-level output (BFS order)
📦
The Queue is the KEY: In Level Order / BFS, the queue ensures we process all nodes at depth d BEFORE any node at depth d+1. This is because queue is FIFO — first in, first out. The nodes we enqueue (children) will be processed after all currently queued nodes (siblings of parent).
Lesson 6 · Implementation

Count Nodes, Height & Leaves

Recursive and iterative approaches for 3 critical operations.

Count Nodes — Recursive
C++
int countNodes(TreeNode<int>* root) {
    if (root == NULL) return 0;  // edge case

    int ans = 1;  // count the current node
    for (int i = 0; i < root->children.size(); i++) {
        ans += countNodes(root->children[i]);  // add children's counts
    }
    return ans;
}
Height of Tree — Recursive
C++
int height(TreeNode<int>* root) {
    if (root == NULL) return 0;  // edge case

    int mx = 0;  // track max height among children
    for (int i = 0; i < root->children.size(); i++) {
        mx = max(mx, height(root->children[i]));
    }
    return mx + 1;  // +1 for current node's edge to its parent
}
📏
Why mx + 1? The height of a node = the maximum height of any child + 1 (for the edge connecting this node to that child). A leaf has no children → max=0 → height = 0+1 = 1. Wait — the slides use "number of edges to deepest leaf" so a leaf = 0 (no edges below it). Adjust the base case to return -1 if using edge-counting.
Count Leaf Nodes — Recursive & Iterative
C++ — Recursive
int countLeafNodes(TreeNode<int>* root) {
    if (root == NULL) return 0;  // edge case

    if (root->children.size() == 0) return 1;  // IT IS a leaf

    int ans = 0;
    for (int i = 0; i < root->children.size(); i++) {
        ans += countLeafNodes(root->children[i]);
    }
    return ans;
}
C++ — Iterative (BFS)
int countLeaves(TreeNode<int>* root) {
    queue<TreeNode<int>*> q;
    q.push(root);
    int leaves = 0;

    while (!q.empty()) {
        TreeNode<int>* f = q.front();
        q.pop();

        if (f->children.size() == 0) {  // leaf check
            leaves++;
        }
        for (int i = 0; i < f->children.size(); i++) {
            q.push(f->children[i]);
        }
    }
    return leaves;
}
Lesson 7 · Binary Trees

Binary Tree Basics

Max 2 children per node. The most important tree type in CS.

Definition & Properties

A Binary Tree is a tree where every parent node has at most 2 children — a left child and a right child.

Max nodes at level i
2i nodes. Level 0 = 1, Level 1 = 2, Level 2 = 4...
Max nodes at height h
2h+1 - 1 total nodes (sum of geometric series)
Min nodes at height h
h + 1 nodes (a straight line — degenerate)
Min height for n nodes
⌊log₂(n)⌋ — when tree is maximally full
BTNode Class in C++
C++ — BTNode
template<typename T>
class BTNode {
public:
    T data;
    BTNode* left;   // pointer to left child (or NULL)
    BTNode* right;  // pointer to right child (or NULL)

    BTNode(T data) {
        this->data = data;
        left = NULL;
        right = NULL;
    }

    ~BTNode() {
        delete left;   // recursively deletes left subtree
        delete right;  // recursively deletes right subtree
    }
};

// Creating a simple tree manually:
int main() {
    BTNode<int>* root = new BTNode<int>(1);
    BTNode<int>* n1   = new BTNode<int>(2);
    BTNode<int>* n2   = new BTNode<int>(3);
    root->left  = n1;
    root->right = n2;
    delete root;  // frees entire tree via destructor chain
}
Level-Wise Input for Binary Tree
C++ — takeInputLevelWise (BT)
BTNode<int>* takeInputLevelWise() {
    int rootData;
    cout << "Enter root data: ";
    cin >> rootData;
    BTNode<int>* root = new BTNode<int>(rootData);

    queue<BTNode<int>*> q;
    q.push(root);

    while (!q.empty()) {
        BTNode<int>* f = q.front();
        q.pop();

        int leftData;
        cout << "Left child of " << f->data << " (-1 = none): ";
        cin >> leftData;
        if (leftData != -1) {  // -1 signals no child
            BTNode<int>* child = new BTNode<int>(leftData);
            q.push(child);      // 1. Create
            f->left = child;     // 2. Connect to parent
        }

        int rightData;
        cout << "Right child of " << f->data << " (-1 = none): ";
        cin >> rightData;
        if (rightData != -1) {
            BTNode<int>* child = new BTNode<int>(rightData);
            q.push(child);
            f->right = child;
        }
    }
    return root;
}
// Convention: enter -1 to indicate NULL (no child)

Tree Traversals

Inorder, Preorder, Postorder — mastered with mnemonics and code.

Inorder (LDR)
Left → Root → Right
Gives SORTED output for BST
4→2→5→1→6→3→7
Preorder (DLR)
Root → Left → Right
Used to COPY a tree
1→2→4→5→3→6→7
Postorder (LRD)
Left → Right → Root
Used to DELETE a tree
4→5→2→6→7→3→1
Mnemonic: The letter position tells you where the Data (root) goes: inDorder = middle, DLR preorder = first, LRD postorder = last.
Complete Traversal Code
C++ — All 3 Traversals
// Inorder: Left → Root → Right
void inorderTraversal(Node* node) {
    if (node == NULL) return;
    inorderTraversal(node->left);
    cout << node->data << "->";
    inorderTraversal(node->right);
}

// Preorder: Root → Left → Right
void preorderTraversal(Node* node) {
    if (node == NULL) return;
    cout << node->data << "->";
    preorderTraversal(node->left);
    preorderTraversal(node->right);
}

// Postorder: Left → Right → Root
void postorderTraversal(Node* node) {
    if (node == NULL) return;
    postorderTraversal(node->left);
    postorderTraversal(node->right);
    cout << node->data << "->";
}

/* For tree: 1->(12,9), 12->(5,6)
   Inorder:   5->12->6->1->9
   Preorder:  1->12->5->6->9
   Postorder: 5->6->12->9->1  */
Depth-Level Traversal (General Tree — Inorder variant)

For a general tree (not binary), "inorder" visits all children except the last, then prints the node, then visits the last child.

C++ — General Tree Inorder
void inorder(TreeNode<int>* root) {
    if (root == NULL) return;

    int total = root->children.size();

    // Visit all children EXCEPT the last
    for (int i = 0; i < total - 1; i++)
        inorder(root->children[i]);

    // Print current node
    cout << root->data << " ";

    // Visit the last child
    inorder(root->children[total - 1]);
}
TraversalOrderKey Use CaseTime
InorderL → D → RGet sorted sequence from BSTO(n)
PreorderD → L → RCopy tree, serialize treeO(n)
PostorderL → R → DDelete tree, evaluate expression treeO(n)
Level OrderBFS by levelFind shortest path, level queriesO(n)
Lesson 9 · Binary Trees

Level Order Traversal

BFS on a binary tree — using a queue.

Level Order for Binary Tree
C++ — BFS / Level Order
void levelOrder(Node* node) {
    if (!node) return;

    queue<Node*> q;
    q.push(node);

    while (!q.empty()) {
        Node* current = q.front();
        q.pop();
        cout << current->data << " ";

        if (current->left)
            q.push(current->left);
        if (current->right)
            q.push(current->right);
    }
}

// Tree: 25 at root
// Output (level by level): 25 15 50 10 22 35 70 4 12 18 24 31 44 69 90
🔄
Algorithm walkthrough: Push root. Loop: pop front, print it, push its left child (if exists), push its right child (if exists). The FIFO queue guarantees we process level 0 before level 1, level 1 before level 2, etc.

Delete Tree & Destructors

Memory management — postorder is the only safe deletion order.

Why Postorder for Deletion?

You must delete children BEFORE the parent. If you delete the parent first, you lose the pointers to the children → memory leak. Postorder (L → R → Root) ensures children are always deleted before their parent.

C++ — deleteTree (General)
void deleteTree(TreeNode<int>* root) {
    if (root == NULL) return;

    // First: recursively delete ALL children
    for (int i = 0; i < root->children.size(); i++) {
        deleteTree(root->children[i]);
    }

    // Then: delete this node
    delete root;
}

// Destructor does the same automatically:
// ~TreeNode() { for each child: delete child[i]; }
// delete root → triggers destructor → cascades down
⚠️ Common mistake: Deleting root first, then trying to access children. Once the node is deleted the pointer is dangling — undefined behaviour. Always Children → Then Parent.
Destructor Cascade Example

For a tree with root=1, children=[2,3], where 2 and 3 are leaves:

  1. delete root (1) triggers ~TreeNode()
  2. Destructor of 1 iterates children → calls delete children[0] (=2)
  3. Node 2 is a leaf, its destructor does nothing, 2 is freed
  4. Back to 1's destructor → calls delete children[1] (=3), freed
  5. 1's destructor finishes → node 1 is freed
Lesson 11 · BST

Binary Search Tree

The ordered binary tree — search in O(log n).

BST Definition & Properties

A Binary Search Tree (BST) is a binary tree with an ordering property:

Left Subtree
Contains ONLY nodes with keys LESS THAN the node's key.
Right Subtree
Contains ONLY nodes with keys GREATER THAN the node's key.
Both Subtrees
Must EACH also be valid BSTs (recursive definition).
No Duplicates
Standard BST has no duplicate keys.
⚡
Why O(log n)? At each node comparison, you eliminate an entire half of the remaining tree. Start with n nodes → n/2 → n/4 → ... → 1. That's log₂(n) steps. Same reason binary search on sorted arrays is fast — and the BST is essentially a sorted structure arranged in tree form.
BST Search — Recursive & Iterative
C++ — Search (Recursive)
Node* search(Node* root, int key) {
    // Base case: not found OR found
    if (root == NULL || root->key == key)
        return root;

    if (root->key < key)            // key is in right subtree
        return search(root->right, key);

    return search(root->left, key);  // key is in left subtree
}
C++ — Search (Iterative — no call stack)
Node* search(Node* root, int key) {
    while (root != NULL) {
        if (root->data == key) return root;      // found
        else if (key > root->data) root = root->right;  // go right
        else root = root->left;                        // go left
    }
    return NULL;  // not found
}
Min & Max in BST
C++ — Min & Max
// Minimum: go as far LEFT as possible
int minNode(Node* root) {
    if (root == NULL) return -1;
    while (root->left != NULL)
        root = root->left;
    return root->data;
}

// Maximum: go as far RIGHT as possible
int maxNode(Node* root) {
    if (root == NULL) return -1;
    while (root->right != NULL)
        root = root->right;
    return root->data;
}

// Both are O(h) where h = height of tree
// Balanced BST: O(log n)   Skewed BST: O(n)
Lesson 12 · BST

BST Insert & Full Program

Inserting while maintaining BST ordering property.

Insert into BST

Insertion follows the same logic as search: navigate left or right based on comparison, insert at the first NULL position found.

C++ — Full BST Program
#include <iostream>
#include <queue>
using namespace std;

struct Node {
    int data;
    Node* left;
    Node* right;
};

Node* createNode(int data) {
    Node* n = new Node();
    n->data = data;
    n->left = n->right = NULL;
    return n;
}

// Insert maintaining BST property
Node* insert(Node* node, int key) {
    if (node == NULL) return createNode(key);  // insert here

    if (key < node->data)
        node->left  = insert(node->left,  key);  // go left
    else if (key > node->data)
        node->right = insert(node->right, key);  // go right
    // if key == node->data: do nothing (no duplicates)
    return node;
}

void inorder(Node* root) {
    if (root == NULL) return;
    inorder(root->left);
    cout << root->data << " ";
    inorder(root->right);
}

int main() {
    Node* root = createNode(10);
    root->left  = createNode(11);
    root->left->left = createNode(7);
    root->right = createNode(9);

    cout << "Before: "; inorder(root); cout << endl;

    root = insert(root, 12);

    cout << "After:  "; inorder(root); cout << endl;
    // Output Before: 7 11 10 9
    // Output After:  7 11 12 10 9
    return 0;
}
OperationAverage CaseWorst Case (Skewed)
SearchO(log n)O(n)
InsertO(log n)O(n)
DeleteO(log n)O(n)
Min/MaxO(log n)O(n)
Inorder (sorted)O(n)O(n)
⚠️ BST worst case: If you insert already-sorted data (1,2,3,4,5,...) into a BST, it degenerates into a linked list! Every node has only a right child. Search becomes O(n). This is why AVL trees and Red-Black trees were invented.
Lesson 13 · Types

Types of Binary Trees

8 types — definitions, properties, use cases.

1. Binary Search Tree
Left < Root < Right. No duplicates. Search in O(log n).
2. Full Binary Tree
Every node has 0 or 2 children. No node with exactly 1 child. Also called "Proper" or "Strict".
3. Perfect Binary Tree
All internal nodes have 2 children AND all leaves are at the same level. Every level is completely filled.
4. Complete Binary Tree
All levels filled EXCEPT possibly the last, which is filled left to right. Nodes as far left as possible.
5. Heap
Complete binary tree + heap property. Max-Heap: parent ≥ children. Min-Heap: parent ≤ children. Used in priority queues & heapsort.
6. Degenerate Tree
Each parent has exactly ONE child. Looks and performs like a linked list. O(n) for all operations.
7. Skewed Tree
Special degenerate tree: left-skewed (all left children) or right-skewed (all right children).
8. Balanced Binary Tree
For every node: |height(left) - height(right)| ≤ 1. AVL tree is a self-balancing BST. Used in databases & search engines.
Hierarchy: Which includes which?

Perfect ⊂ Complete ⊂ Binary Tree

Perfect ⊂ Full ⊂ Binary Tree

Heap = Complete + Heap Property

BST = Binary Tree + Ordering Property

AVL Tree = BST + Balanced Property

📐
Perfect vs Complete vs Full — the classic confusion:
Perfect: ALL leaves same level, ALL internal nodes have 2 children.
Complete: Last level may be incomplete but filled LEFT TO RIGHT.
Full: Every node has 0 or 2 children (NOT necessarily same height).
A Perfect tree is also Complete and Full. A Complete tree is not necessarily Full.
Lesson 14 · Exam Prep

🎯 Exam Q&A

Likely exam questions with full answers. Click to reveal.

📋 Theory Questions
Q1
Define a tree. How is it different from a graph?
A tree is a connected, acyclic hierarchical data structure where each node has exactly one parent (except the root which has none). Unlike a general graph, a tree: (1) has no cycles, (2) is connected (all nodes reachable from root), (3) has exactly n-1 edges for n nodes, and (4) has a unique path between any two nodes.
Q2
What is the difference between height and depth of a node?
Depth = number of edges from ROOT to the node (root depth = 0). Height = number of edges from the node DOWN to its deepest descendant leaf (leaf height = 0). Height of the tree = height of the root = depth of the deepest leaf.
Q3
Why is a queue used for level order traversal?
Level order traversal is essentially BFS (Breadth-First Search). A queue's FIFO (First In, First Out) property ensures we fully process all nodes at level d before moving to level d+1. We enqueue children when visiting a parent — those children get processed only after all current-level nodes are done.
Q4
What is the maximum and minimum number of nodes in a binary tree of height h?
Maximum = 2^(h+1) - 1 (when every level is completely filled — a perfect binary tree). Minimum = h + 1 (when every internal node has only one child — a degenerate/skewed tree).
Q5
List all properties of a Binary Search Tree.
(1) Left subtree contains only keys LESS THAN root's key. (2) Right subtree contains only keys GREATER THAN root's key. (3) Both left and right subtrees are themselves valid BSTs. (4) No duplicate keys. (5) Inorder traversal of a BST always gives sorted (ascending) output.
Q6
Differentiate: Full Binary Tree vs Complete Binary Tree vs Perfect Binary Tree.
Full: every node has 0 or 2 children (no node with exactly 1 child). Complete: all levels filled except possibly the last, which is filled left to right. Perfect: every internal node has exactly 2 children AND all leaves are at the same level. A Perfect tree is both Full and Complete, but not vice versa.
Q7
Why should tree deletion use postorder traversal?
In postorder (Left → Right → Root), children are always deleted before their parent. If we delete the parent first, we lose the pointers to the children, causing a memory leak. Postorder guarantees the node is deleted only AFTER all its children and their subtrees are safely freed.
Q8
What is a degenerate tree? When does a BST become degenerate?
A degenerate tree is one where each parent has only one child, making it structurally identical to a linked list. A BST becomes degenerate when elements are inserted in sorted (or reverse sorted) order — e.g., inserting 1, 2, 3, 4, 5 creates a right-skewed tree. All operations degrade to O(n).
💻 Code/Algorithm Questions
Q9
Write a recursive function to count the number of leaf nodes in a binary tree.
int countLeaf(Node* root) { if (!root) return 0; if (!root->left && !root->right) return 1; return countLeaf(root->left) + countLeaf(root->right); } — Base case 1: NULL → 0. Base case 2: both children NULL → it's a leaf, return 1. Recursive case: sum of leaf counts from left and right subtrees.
Q10
Trace the inorder traversal of: root=25, left=15, right=50, 15's children={10,22}, 50's children={35,70}.
Inorder (L→D→R): Go left to 15 → go left to 10 → 10 has no left → print 10 → 10 has no right → back to 15 → print 15 → go right to 22 → print 22 → back to 25 → print 25 → go right to 50 → go left to 35 → print 35 → print 50 → go right to 70 → print 70. Result: 10, 15, 22, 25, 35, 50, 70 (sorted!)
Q11
What is the time complexity of searching in a BST, and when does it become O(n)?
Average case: O(log n) for a balanced BST because each comparison eliminates half the remaining nodes. Worst case: O(n) when the tree is skewed/degenerate (inserted in sorted order). This is why balanced BSTs (AVL, Red-Black) are preferred in practice — they guarantee O(log n) always.
Q12
Explain how a destructor can be used to free a tree's memory in C++.
Define the destructor ~TreeNode() to iterate through the children vector and call "delete child[i]" for each child. This recursively triggers each child's destructor before that child is freed. When you call "delete root" at the end, this cascade ensures the ENTIRE tree is freed bottom-up (postorder), preventing memory leaks.
🧮 Calculation Questions
Q13
A perfect binary tree has height 4. How many total nodes does it have?
Formula: 2^(h+1) - 1 where h = height. Answer = 2^(4+1) - 1 = 2^5 - 1 = 32 - 1 = 31 nodes.
Q14
Given a binary tree with 100 nodes, what is the minimum possible height?
Minimum height = ⌊log₂(n)⌋ = ⌊log₂(100)⌋ = ⌊6.64⌋ = 6. This is achieved when the tree is as complete (as "bushy") as possible.
